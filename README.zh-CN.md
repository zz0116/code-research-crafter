## 🎯 什么是 Code Research Crafter？

**Code Research Crafter** 是一个综合性的技能，提供结构化的 6 阶段工作流，用于深度代码库研究和专业提案撰写。它通过基于证据的技术写作，弥合代码分析与社区参与之间的鸿沟。

## ✨ 核心特性

- **🔍 系统化代码分析** - 通过结构化方法论探索和理解复杂代码库
- **📚 学术研究集成** - 查找和引用相关论文、算法和现有技术
- **👥 社区情报** - 分析 GitHub Issues、讨论和维护者反馈
- **🏗️ 架构设计** - 创建分层解决方案，包含数据模型和分阶段实施计划
- **📝 专业文档生成** - 生成多语言结构化技术文档
- **🚀 RFC 发布** - 为开源社区撰写和提交专业 RFC

## 🔄 6 阶段工作流

### 阶段 1：问题发现与代码分析
识别目标领域，执行深度代码分析，并用量化指标记录发现。

### 阶段 2：学术与社区研究
搜索学术论文，提取关键见解，分析 GitHub 社区讨论。

### 阶段 3：解决方案设计
定义设计原则，构建分层解决方案，设计数据模型，规划实施阶段。

### 阶段 4：文档生成
生成具有适当结构和引用的综合技术文档。

### 阶段 5：英文 RFC 撰写
按照开源社区标准撰写专业 RFC Markdown 文档。

### 阶段 6：GitHub 发布
以适当格式和社区参与度提交 RFC 到 GitHub。

## 📖 何时使用此技能

在以下情况下使用 Code Research Crafter：

- ✅ 分析开源代码库并提出增强建议
- ✅ 以学术严谨性研究技术问题
- ✅ 基于证据决策设计系统架构
- ✅ 为开源社区创建专业 RFC
- ✅ 用适当引用记录复杂技术提案

## 🎨 设计原则

1. **基于证据的分析** - 引用具体代码位置并量化问题
2. **学术严谨性** - 引用近期论文（2024-2025）并正确标注来源
3. **以人为本的设计** - 使用组织类比，设计渐进式采用方案
4. **成本意识** - 跟踪 Token/性能影响，设计高效方案
5. **社区参与** - 引用现有问题并邀请协作

## 🛠️ 工具与资源

此技能利用各种工具进行全面研究：

- **代码分析**：`glob`、`grep`、`read`
- **学术研究**：`WebSearch`、`WebFetch`
- **文档生成**：`python-docx` 用于专业文档生成
- **发布**：`browser_use_desktop` 用于 GitHub 提交
- **版本控制**：`desktop_terminal_execute` 用于 Git 操作

## 🚀 快速开始

1. **确定目标** - 选择要研究的代码库或技术问题
2. **遵循工作流** - 系统地完成 6 个阶段

4. **参与社区** - 分享你的 RFC 并收集反馈

## 📋 常见模式

### 分层架构
```
Layer 1: 基础层（数据收集）
Layer 2: 增强层（基于 L1 构建）
Layer 3: 智能层（基于数据的 AI/ML）
Layer 4: 治理层（控制/监控）
```

### 双轨数据
- **用户定义（静态）**：配置、提示词
- **系统学习（动态）**：指标、模式

### 可见性层级
- `private`：仅所有者
- `team`：协作者
- `global`：所有用户
